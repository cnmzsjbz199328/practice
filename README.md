# Stickman Animator

JSON-driven SVG stickman animations. Hand-write the JSON or pose visually in the editor;
preview live with [Remotion](https://www.remotion.dev/); render to MP4 with a single
`npm run render` command.

Live editor: [practice.tj15982183241.workers.dev](https://practice.tj15982183241.workers.dev/)
(deployed on Cloudflare Workers Static Assets).

## Pipeline

```
                     ┌──────────────┐
                     │ animation.json │
                     └──────┬─────────┘
                            │
                ┌───────────┴────────────┐
                ▼                        ▼
       ┌────────────────┐       ┌────────────────────┐
       │ Web editor     │       │ npm run render     │
       │ (Remotion      │       │ (Remotion CLI +    │
       │  Player)       │       │  headless Chrome)  │
       └────────────────┘       └─────────┬──────────┘
                                          ▼
                                     output.mp4
```

JSON is the single source of truth — it can be hand-written, generated by a script,
or saved by the editor. Both the live preview and the MP4 renderer read the same shape.

## Quick start

```bash
npm install
npm run dev                                          # editor at http://localhost:5173
npm run render -- examples/wave.json out/wave.mp4    # render the wave example
npm run render:wave                                  # shortcut for the line above
```

The first `npm run render` downloads ~200 MB of headless Chromium into
`~/.cache/remotion`. Subsequent renders are fast.

## Editor

- Drag joint handles to pose the figure.
- Click an empty area on the timeline to add a keyframe; click a keyframe to select.
- Use the Project panel to **Save JSON** / **Load JSON** / load a built-in example.
- Use the Animation panel to set duration, easing, loop.
- Visit `?animation=<URL>` to load a JSON over HTTP on cold load.

The editor's Remotion `<Player>` shares its `interpolatePose` function with the CLI,
so what you preview is what gets rendered.

## JSON schema (v1)

```ts
{
  version: 1,
  canvas: { width: number, height: number, fps?: number = 30 },
  bones: Array<{
    id: string,
    parent: string | null,
    length: number,
    isHead?: boolean,
  }>,
  keyframes: Array<{
    id: string,
    time: number,                                // seconds
    pose: {
      rootX: number, rootY: number,
      angles: Record<string, number>,            // radians, parent-relative, keyed by bone id
    },
  }>,
  animationParams: {
    durationSeconds: number,
    easing:
      | { kind: 'linear' }
      | { kind: 'ease-in-out' }
      | { kind: 'cubic-bezier', p1x: number, p1y: number, p2x: number, p2y: number },
    loop: boolean,
  },
}
```

See `examples/wave.json`, `walk.json`, `jump.json` for full samples.

## CLI flags

```
npm run render -- <input.json> <output.mp4> [--composition=Stickman] [--quality=low|default|high]
```

- `--composition` — Remotion composition id (only `Stickman` exists today).
- `--quality` — maps to ffmpeg CRF: `low=28`, `default=23`, `high=18`.

The script Zod-validates the JSON before bundling, so syntax errors surface immediately
instead of as opaque webpack failures.

## Stack

- React 18 + TypeScript + Vite 6
- Tailwind CSS + DaisyUI
- Zustand for editor state
- Zod for schema validation
- Remotion (`@remotion/player` for preview, `@remotion/bundler` + `@remotion/renderer`
  for CLI rendering)

## Architecture notes

- **Forward kinematics**: each bone is parent-relative (length + angle). Bone lengths
  stay constant for free; joint dragging is a one-bone `atan2`. See
  `src/skeleton/forwardKinematics.ts`.
- **Shared interpolation**: `src/skeleton/interpolate.ts` is called by both the
  Remotion composition (per-frame) and the editor (preview). Iterates whatever bone
  ids the document defines, so custom skeletons work end-to-end.
- **Bundler purity**: `src/skeleton/canvas.ts` holds canvas constants so
  `src/remotion/Root.tsx`'s import graph stays DOM-free. Remotion's webpack bundler
  never pulls in pointer-event handler code.
- **Cloudflare deploy**: only the Vite build (`dist/`) ships to CF Workers. The CLI
  renderer is dev-only (`@remotion/cli/bundler/renderer`, `tsx` are devDependencies).

## Alternate output: standalone SMIL SVG

The editor's "Export SMIL SVG" button still works. It produces a `.svg` file with
native `<animate>` tags that animates itself in any browser without CSS or JS — the
original MVP path. Useful for embedding the animation in pages where MP4 is
overkill. Remotion is the primary export path; SMIL is kept because it's free.

## Out of scope

- MP4 rendering on Cloudflare (Workers can't run Chromium). Use a local CLI or
  add Remotion Lambda later.
- Importing existing SVGs as bones.
- Multi-character scenes, undo/redo, persistence beyond JSON files.
